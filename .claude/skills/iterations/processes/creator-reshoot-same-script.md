# Process — Creator Reshoot, Same Script

Send the proven script to a new creator. Message stays identical; only the face delivering it changes. The script is the hardest part of an ad to nail, and this iteration treats it as solved — the test is whether a new creator unlocks a new audience or refreshes a fatiguing one without breaking what is already converting.

## Required inputs

- The winning ad and its performance data.
- The full script of the winning ad, broken into beats with timing.
- The brand's ICP and personas. The new creator must match either the existing target archetype (fresh face, same demographic) or a deliberately different one (new demographic to expand reach).
- The brand's creator-fit principles, if defined — tone, age range, energy level, aesthetic.
- Brand context for compliance — any creator must be able to deliver every line of the script without violating brand voice or compliance rails.

## Pre-check

- Is the script clearly the working element? If the hook is the only thing working and the script underneath is weak, reshooting will not save it.
- Has the current creator clearly fatigued, or is the brand trying to expand to a new demographic? Either reason is valid. The recommendation should name which one.
- Does the brand have access to a creator pool that fits? If casting is the constraint, flag it in the recommendation.

## Decision: same demographic or new demographic

This is the single most important choice in this process. Make it deliberately.

- **Same demographic, fresh face.** Use when the script is converting and the current creator is fatiguing within the existing target audience. Goal is to refresh the visual without changing who the algorithm targets.
- **Different demographic.** Use when the brand is trying to reach a new archetype with proven messaging. Goal is incremental spend by unlocking a new audience segment.

Do not pick a "new demographic" as the default. If the brand has not exhausted the same-demographic-fresh-face variant, that one is lower risk.

## Execution

1. **Surface the proven script verbatim.** Include every line, every beat, every CTA. Note timing per beat so the reshoot matches the proven pacing.

2. **Define the creator archetype precisely.** Age range, gender presentation if applicable, tone, energy level, lifestyle signal, and any specifics that match the brand's ICP and the script's delivery requirements. Vague briefs produce off-target reshoots.

3. **Identify what must stay constant.** Script, structure, pacing, key visual beats, CTA. Be explicit so the team does not let the new creator "make it their own" in ways that drift from the proven format.

4. **Identify what is allowed to change.** Wardrobe, setting, b-roll cutaways, lighting if it does not change brand tone, micro-expressions. These are the variables that produce a meaningfully different ad without changing what is being tested.

5. **Note compliance for the new creator.** If the script contains claims that require specific credentials (medical, professional, financial), the new creator must qualify. Do not propose reshooting a doctor's script with a non-credentialed creator.

## Output content for this iteration

- A name that signals what is changing. Example pattern: "Same-Script Reshoot — [Target Demographic Description] Creator."
- The verbatim script that the reshoot uses, with beats and timing.
- The creator archetype with specifics.
- The decision rationale — same demographic for fresh face, or different demographic for new audience reach — tied to the diagnosis.
- The "do not change" list. Script, structure, pacing, CTA, key visual beats.
- The "allowed to change" list. Wardrobe, setting, b-roll, lighting nuance.
- The proven-pattern note: a high-performing script delivered by a new creator is one of the consistently strongest iteration paths because every learning from the original is preserved while the visual is refreshed.

## What never to do

- Never reshoot a script that is not the working element. If the message is the bottleneck, this process is the wrong choice.
- Never let the new creator rewrite the script. This iteration is testing the face, not the words.
- Never propose a creator outside the brand's tone or compliance fit, even if their reach is appealing.
- Never combine a creator reshoot with other changes in the same recommendation. One variable per iteration so the test produces a clean learning.
