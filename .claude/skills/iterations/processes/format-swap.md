# Process — Format Swap

Take a winning message and rebuild it in a different format. The signal-change ladder identifies format as the loudest lever Parker can pull — moving a winning concept from video to static, static to video, video to carousel, or any other cross-format move fundamentally changes how the algorithm classifies and serves the ad. The message is the proven part; the format change is the test.

## Required inputs

- The winning ad and its performance data.
- The core message of the winning ad, stripped down to its single most powerful claim, insight, or moment. This is what the new format must carry. If the message is not identifiable as a single distilled idea, do not run this process — the message is not yet sharp enough to translate.
- The target format. This is a deliberate choice, not "let's try every format." Pick the format that the brand has not yet tested with this concept and that fits the audience and placement strategy.
- Brand context including compliance, voice, and any platform considerations.

## Decision: which target format

This is the choice that determines the value of the iteration. Name the target format by its real tag from `parker-system/creative-strategy-context/ad-formats/` rather than an invented blend, so the recommendation speaks the same language the account uses to classify creative. Common moves:

- **Video to static.** Distill the most powerful moment from the video into a single headline-plus-visual that converts at a glance — a `Headline Only`, `Headline + Benefits`, or `Native` static depending on the brand's static playbook. Strongest when the brand has a winning video but limited static creative in rotation.
- **Static to video.** Rebuild the winning static as video using the winning headline as the opening hook. The natural target tags are `Motion Videos` if the static is animated into motion, or a creator-led `UGC Single` or `Demo` if it is rebuilt as a shot ad. Strongest when the brand has a winning static and wants to extend the message into higher-reach video placements.
- **Video to carousel.** Convert the video argument into a `Carousel` — a multi-frame swipeable. Strongest when the message has a natural problem-solution-proof structure.
- **Video to motion.** Convert to a looping `Motion Videos` piece — a designed static animated into motion, usually under ten seconds. Strongest when the message can land in three to five seconds without dialogue.
- **Static to carousel.** Expand a single static into a `Carousel` story. Strongest when there are multiple proof points or use cases to surface that one static cannot carry.
- **Trend conversion.** Adapt the message into the `Trends` format — a live viral audio, transition, or platform template — or a `Meme` static. Strongest when the brand voice supports it and the trend is live.

## Pre-check

- Is the message itself proven? If not, format swap is the wrong move — fix the message first.
- Has the brand already tested this format with this concept? If yes, pick a different format or a different process.
- Is the message portable? Some messages are inseparable from the format that delivered them. A long emotional founder story does not become a strong six-second static. Be honest about portability before recommending the swap.

## Execution

### When moving from video to static

1. Identify the single most powerful moment in the video — the moment of strongest emotional charge, the clearest proof, or the most quotable line.
2. Distill that moment into a headline. The headline must work as a cold-audience opener; if it requires the prior video context to make sense, it is not yet right.
3. Choose the visual. Either a stylized still from the video, a hi-fi product shot, or a creator lifestyle frame — whatever matches the brand's static playbook.
4. Specify the supporting copy, the CTA, and the layout. Apply the static headline iteration playbook for the headline structure if needed.

### When moving from static to video

1. The winning headline becomes the opening hook of the video — text overlay in the first frame, voiceover delivering it, or both.
2. Build a body that delivers on the headline's promise. The body can borrow from the brand's other winning video ads if the tone matches, or be a fresh write from the same script principles.
3. Specify the CTA, the pacing for the target audience age range, and any visual proof beats.

### When moving from video to carousel

1. Card 1 carries the hook or the strongest moment from the video.
2. Subsequent cards each carry one proof point, one objection handler, or one benefit translation — pulled directly from the video script.
3. Final card carries the CTA.
4. Specify each card's headline, supporting copy, and visual.

### When moving from static to carousel

1. Card 1 keeps the winning headline and visual.
2. Subsequent cards expand into proof points, use cases, or variants that the static could not show.
3. Final card carries the CTA.

### When moving to motion or GIF

1. Identify the core three-to-five-second visual loop that carries the message.
2. Specify the text overlay, the loop point, and what the viewer sees on each pass.

### When moving to a trend format

1. Identify the specific trend by name and the structure it requires.
2. Map the winning message into that structure exactly. The trend's format is fixed; only the message variables are filled in.
3. Specify timing, the audio if the trend requires it, and the visual treatment.

## Output content for this iteration

- A name that signals the move. Example pattern: "Format Swap — Video Hook to Static Headline."
- The exact core message being carried over, distilled to one or two sentences.
- The exact element from the source ad being translated, with timestamp or copy reference.
- The exact target format spec — headline, visual, supporting copy, CTA, structure, pacing.
- The transition rationale — why this message survives the format change.
- The proven-pattern note: format is the loudest signal lever; this iteration leverages a proven message into a fundamentally different algorithmic classification.

## What never to do

- Never compress an entire video argument into a single static. Distill, do not compress. Pick one powerful moment, do not try to fit the whole video.
- Never run a format swap on a message that is not yet proven. Format swap is for amplifying winners, not finding them.
- Never propose a format swap that has been tested. Check iteration history.
- Never let the format change drag the message off the proven version. The message is fixed; only the format moves.
