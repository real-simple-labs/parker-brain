# Deliverable Specs

The output spec depends on the deliverable type set in Step Zero Question 4. **The shape of the deliverable changes the shape of the work** — a hooks-only brief should not get a script and storyboard, and a shot-list audit should not get a script at all.

Read this before producing anything. Padding an output with sections the user did not ask for wastes their time and is a named failure of this skill.

Every deliverable type below closes with the **Launch Context Summary** and the two gate blocks, per the output structure in `SKILL.md`. Those are constant across all types.

## Default — full script plus storyboard

The default when Question 4 was not specified. Runs through the `scriptwriting` skill, whose output contract governs the script itself; what follows is the launch-specific shape layered on it.

- **Overview.** One to three sentences. The launch scenario, the phase (warm or cold), the persona, the awareness stage, and the core concept. Producer-readable. No framework labels, sophistication levels, or internal strategy vocabulary.
- **Script.** Full script as a clean vertical block. One line per spoken or text-overlay beat. No labels, no segment numbers, no shot directions, no on-screen-text callouts mixed in. If the format is text-on-screen only with no voiceover, note that at the top of the section and present the overlay copy line by line.
- **Script with storyboard.** The same script in a table with three bolded columns: **Script | Shot | On-Screen Text.** One row per moment, first row labeled "Hook:". For text-on-screen videos, replace the Script column with On-Screen Text and drop the third column, leaving two: **On-Screen Text | Shot.**

Where `scriptwriting`'s contract asks for plain-sentence reasoning ("Why this script"), that governs — fold the scenario and awareness stage into the reasoning where they explain the call, and let the Launch Context Summary carry the tags. Do not produce a form-shaped list of labels where plain sentences were asked for.

## Hooks only

Five to ten hooks tied to angle territories. Note the angle category and awareness stage for each. **No full scripts.**

Runs through the `hooks` skill for the writing craft; the launch layer supplies the angle territories from the scenario playbook and the newness lever each hook has to make visible.

## Ad copy and headlines only

Typically for static ads paired with hero imagery. Copy variants plus headlines plus suggested CTAs, **one block per angle.** Note the persona and awareness stage per block. Include one or two hook lines that can double as headlines.

Runs through the `headlines` skill for the craft.

## Founder or talking-head video

Script in the founder's voice with light shot direction — setting, framing, energy. **No full storyboard table**; the format is long-take and direct-to-camera, and a beat-by-beat storyboard misrepresents how it gets shot.

Note the founder POV and **the one concrete thing they are saying that only the founder can say.** If that thing cannot be named, the deliverable does not need to be a founder video, and something has been mistaken for authenticity that is really just a person on camera.

## Shot list

Bullet shot list aligned to angle territories. **No script copy.** Note sourcing requirements — props, locations, talent, wardrobe — since the shot list's job is to make the shoot possible, not to describe the ad.

## Pre-launch audit

A checklist diagnosis of what is missing from the provided shot list or asset set, mapped against:

- Warm versus cold creative covered
- Awareness stages covered
- Scenario fit
- Proof types planned
- Founder versus UGC mix
- Day-one versus evergreen split

This deliverable is a gap analysis, not a production. It answers "what is missing," so it leads with the gaps rather than cataloguing what already exists.

## Multi-video theme or launch content plan

Three parts, in order:

1. **The through-line concept** — the unifying idea.
2. **How each video plays a role inside the through-line** — what job each asset does in the system.
3. **The unifying hook, visual, or copy device** — the mechanical thing that makes the set read as one campaign.

Then a brief per-video role summary so the producer knows what each asset is doing. Do not write the individual scripts; the through-line is the asset. See `special-launch-contexts.md`.

## Format recommendations only

Five to eight formats fit for this scenario, each with a one-line rationale: why this format, for which audience, at which awareness stage. **Do not write scripts.**

Name formats by the account's own AI tag taxonomy, as the diagnostic surfaced them. Inventing blended format names is banned — the tags are shared vocabulary the user sees in their own dashboard.
