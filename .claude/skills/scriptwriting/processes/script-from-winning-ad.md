# Process — Script From Winning Ad

A NEW script built from the brand's own winner DNA. Not a variant of the winner — a fresh script whose hook, body, or proof spine is a validated component lifted from an ad the brand already proved.

The winner is the only reference. Its pieces are already de-risked. The job is to recombine them into something that has not run yet.

## Required inputs

- The brand's winning ad with full transcript, timing, and the performance signal that earned it "winner" status.
- Brand baseline — the brand's existing scripts.
- Brand profile, ICP, voice of customer, compliance constraints.
- The component being carried — which part of the winner is the validated seed.

## Philosophy

A winner is a bundle of proven parts: a hook that stops the scroll, a body that holds attention, a proof beat that converts. Each part can be lifted out and reused on its own. The proven hook can carry a new body. The proven body can be re-opened with a new hook. Two validated pieces from two different winners can be stitched into one Frankenstein script. The validation travels with the piece, so the new script starts ahead.

This is not iteration. Iterations skill produces variants of one winner — same ad, tuned. This produces a new ad from winner parts.

## When to pick

- The brand has at least one ad with real performance signal.
- The user wants something new, not a tuned variant.
- A specific component (the hook, the body, the proof) is the thing worth reusing.

## Execution

### Step 1 — Isolate the validated component
Name which part proved out. The hook that drove the watch-through. The body that held. The proof beat that converted. Pull it verbatim with its timing.

### Step 2 — Pick the recombination move
Three moves. **Hook carry:** keep the proven hook, write a new body and proof under it. **Body re-open:** keep the proven body, write a fresh hook into it. **Frankenstein stitch:** take a validated piece from winner A and a validated piece from winner B, stitch at the seam, write the connective tissue.

### Step 3 — Build the new material around the kept part
Match the brand baseline for the parts you write. The kept part sets the pacing and energy; the new material lands in the same register so the seam disappears.

### Step 4 — Source new language from VoC
The new beats pull from real reviews, comments, surveys. The kept beat already speaks the customer's language; the new beats must too.

### Step 5 — Seam audit
Read it aloud start to finish. The transition between kept and new must be invisible. If the seam shows — a pacing jolt, a voice shift, an energy drop — rewrite the connective tissue until it reads as one script.

## What good looks like

- One clearly named validated component carried into a genuinely new script.
- The seam is invisible on a read-aloud.
- The new material matches the brand baseline and pulls VoC language.
- The script is distinct enough that it is a new test, not a variant.

## Common failures

- Producing a variant instead of a new script. If the result is the winner lightly tuned, route to the iterations skill instead.
- A visible seam — the kept part and new part read as two different ads.
- Carrying the winner's words into the new beats verbatim. Only the named component is kept; everything else is written fresh.
- Stitching a claim the brand cannot defend in the new body. Compliance walls override.

## Cross-route

Variants of a single winner — same ad, tuned hook or angle — belong to the iterations skill, not here. This process only fires when the goal is a NEW script from winner DNA.
