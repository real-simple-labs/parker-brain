# Process — Script From Open Lane

Whitespace-driven. The gaps and whitespace docs name an empty lane the category is not running — the absolution register, the problem-mirror, the explanation hook. The script is built INTO the named lane, with the lane's evidence as the brief.

The lane is empty because no one has run it, and the gaps doc has already argued why it is worth running. The script claims it first.

## Required inputs

- A named open lane with its supporting evidence. Source: the current quarter's gaps read at `audits/[YYYY-Q]/gaps-opportunities-inspo.md`.
- Brand baseline — the brand's existing scripts.
- Brand profile, ICP, voice of customer, compliance constraints.

## Philosophy

A category's ads cluster. Everyone runs the same hooks, the same registers, the same proofs, and the clustering leaves lanes empty. The gaps doc finds those lanes and names them — the absolution register no rival uses, the problem-mirror angle the category avoids, the explanation hook nobody bothers with. An empty lane is an opening: the first brand to run it owns a register the audience has not been pitched in.

The script is built into the lane, not adapted from a reference. There is no reference because the lane is empty — that is the point. The gaps doc's evidence for why the lane is open and worth running is the brief. The discipline is staying in the lane: a script that drifts back toward the category's crowded registers forfeits the whitespace.

## When to pick

- The gaps doc names a specific open lane with evidence behind it.
- The brand can credibly occupy that lane.
- The user wants to run where the category is not, not adapt what it already runs.

## Execution

### Step 1 — Pull the named lane and its evidence
From the gaps doc, pull the specific open lane and the evidence that it is empty and worth running. The evidence is the brief.

### Step 2 — Define the lane's register
Name what makes the lane distinct. The absolution register forgives where the category shames. The problem-mirror reflects the audience's situation back where the category jumps to solution. The explanation hook teaches where the category sells.

### Step 3 — Build the script into the lane
Hook, body, and close all hold the lane's register. There is no skeleton to adapt; the lane's register and the brand baseline set the shape.

### Step 4 — Source language from VoC
Even a net-new lane speaks in customer language. Pull from reviews and comments so the lane is occupied in the audience's own words.

### Step 5 — Lane-fidelity, baseline, and compliance pass
Confirm the script never drifts back into the category's crowded registers. Match the brand baseline's pacing and CTA. Confirm every claim is defensible.

## What good looks like

- The script holds the named lane's register from hook to close.
- The lane's evidence visibly shaped the brief.
- No drift back toward the category's crowded registers.
- Customer language throughout; pacing matches the brand baseline.

## Common failures

- Drifting back into the category's standard register and forfeiting the whitespace.
- Treating the lane name as a label without building the register into the script.
- Picking a lane the brand cannot credibly occupy.
- A claim the brand cannot defend.
