# Strategy — what makes a hypothesis worth running

The roll-up already decided this loop matters. Advancing it well is about one thing: turning a good question into a bet that research can actually settle, without losing what made the loop strategic.

**A prediction is a bet, not a fact.** The loop asked an open question; the hypothesis commits to an answer and goes to test it. That commitment is the point — a prediction Parker is unwilling to be wrong about is not a hypothesis. State the bet plainly, mark it as Parker's, and make sure a real outcome could prove it wrong.

**The strategist's moves carry into the plan.** The same instincts that sharpened the loop shape how it gets tested. Size the opportunity before planning how to capture it. Ground the test in data Parker can reach before reaching for the open web. When the loop mirrored a competitor pattern, the test mirrors it back to the brand's own data. When the loop named an empty lane, the test asks why the lane is empty — tried and failed, blocked by a constraint, or genuinely open — because each answer routes a different next move.

**Triangulate the full picture, don't build a study — the cut is depth, not breadth.** The test is the few complementary sources that each see a different angle — the ad account for what's been run, reviews for whether the demand is real, competitors for whether it works in market — not a single pull and not a six-step study. One source has blind spots: the ad account especially cannot speak to an opportunity the brand never tested, so reviews and competitors are what reveal whether an untested lane is real. The thing to cut is over-engineering any one source into confound-separations and time-series; the thing to keep is the handful of angles that together settle it. A strategist asks "what few reads, together, tell me if this is true and worth it?" — not "what is the one pull," and not "what is everything I could analyze." And a loop that never closes is worse than one Parker honestly marks unvalidatable.

**Build the test to be disprovable.** The reverse-evidence step is not a formality. Most weak validations come from a plan that only looked for confirmation. Decide, before running, what evidence would make the prediction false — and go look for it as hard as for the evidence that it is true.

**Set the bar before you look.** Naming what would count as validated, invalidated, inconclusive, or insufficient — in advance — is what keeps the validation honest. A bar set after the research is a bar moved to fit the finding.

**Recognize the unvalidatable.** Some loops can only be settled by a real-world test Parker cannot run or data the brand has never published. The senior move is to name that plainly and route it to the user, not to dress an unanswerable loop in a plan that looks rigorous and closes nothing.
