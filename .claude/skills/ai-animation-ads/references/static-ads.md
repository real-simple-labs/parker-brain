# Static Ads — AI-Generated Statics Track

Statics are not the side dish: healthy DTC accounts run 30–55% statics, they find winning hooks ~28% cheaper than video, and statics fatigue fast (7–14 days at scale) so volume matters. Every video concept should ship with 2–4 statics derived from its hero frames and hooks; statics also run standalone as the cheap hook-testing layer whose winners graduate into video ads.

## Winning static archetypes (pick 2–3 per batch, diversify axes)

| Archetype | What it is | When it wins |
|---|---|---|
| **Callout / annotated product** | Product hero with arrowed benefit callouts | The supplement/ingredient-story workhorse; TOF/MOF education |
| **Benefit stack** | Product at ~40% canvas over 4–6 outcome bullets ("wake up not sore," not features) | Cheapest to make; retargeting; pre-answers objections |
| **Us-vs-them grid** | Side-by-side vs "the old way" (genericized, never named) | Displacing an incumbent solution |
| **Testimonial card / forum thread** | One SPECIFIC-outcome quote + stars + product corner; or Reddit-style thread screenshot look | Trust gaps MOF/BOF; thread-style routinely beats video on CPA |
| **Reasons-why listicle** | Numbered 3–5 benefit list, text-led | Supplements/meal replacement staple |
| **Offer card** | Bold typography: discount + urgency + trust badges stacked | BOF, promos — urgency must be real |
| **Editorial/advertorial** | Article-style layout, dek headline | Premium brands; native feel |
| **Ugly/lo-fi** | Whiteboard, handwritten, sticky note, MS-Paint | Outperforming polished creative in 2026 (engagement-velocity ranking rewards raw); one benefit, seconds to read |
| **Cartoon/animation-style static** | A hero frame from the animated ad + headline: the villain confessing in a speech bubble, the mechanism cross-section, the payoff scene | Extends every animation concept for near-zero cost; obviously-illustrated = lighter AI-labeling treatment |
| **Comic strip / multi-panel** | 3–4 panel problem→solution arc in the ad's animation style | "Video-level storytelling without production"; products needing narrative explanation |
| **Meme** | Trending format + brand | TOF humor brands; ship fast, decays fast |

**Anatomy rule:** statics carrying 5–6 of these win ~2x more often than those with ≤2 — specific-friction hook · visual subject readable in <0.5s · ONE promise only · social proof as a primary visual element · action-verb CTA · logo visible but non-central (8–12% canvas height).

## Generation workflow

**THE STATICS PIPELINE v2 (seven steps, three review gates — canonical, never skip or reorder):**
1. **Strategy + plan** → **GATE 1: spawn `static-strategy-reviewer`** on the sprint brief. Proceed only on APPROVE; apply its ranked fixes first otherwise.
2. **Copywriting** (the 2.5a language pass + 2.5b headline gate below) → **GATE 2: spawn `static-copy-reviewer`** on the full copy deck. Every line ships A+ or gets its surgical rewrite applied. This gate also owns the AI-ism sweep.
3. **Design psychology + references** (RULE ZERO/ONE/THREE + the reference-fit scorecard below, recreation mechanics per ad) → **GATE 3: spawn `static-design-reviewer`** on the design/reference plan.
4. **Generation plan**: models, reference imports, aspect ratios, batch order under the concurrency cap, cost preflight vs the autonomy budget.
5. **Generate** (waves at the cap; clean-plate + post-set text where formats allow).
6. **Review to 10/10**: re-run all three reviewers on the RENDERED set (design reviewer side-by-side with references, zoomed OCR); the output is a per-ad "path to 10" plan, each item tagged re-roll / post-fix / accept-risk.
6.5. **The convergence loop (RLHF gate — the owner's feedback should never be the first review the work gets)**: spawn `creative-critic` on the final artifacts. It predicts the owner's grade from `references/owner-feedback-model.md` (the graded corpus of every past owner correction) plus the doctrine docs, and raises the objections the owner would raise. Anything below predicted A+ gets its named revision applied and re-graded; max 3 iterations, then deliver best-with-flags. AFTER delivery, any owner correction that still lands is appended to the corpus the SAME session (event, root cause, fix, generalized rule) so the critic never misses it again. This loop is format-agnostic: run it for video, hooks, scripts, and any future format with that format's doctrine.
7. **Make the changes and deliver**: apply every path-to-10 item per the repair doctrine (respell-and-re-roll for label/text zones, deterministic post-fix only on flat empty zones), then deliver with receipts (verdicts, spend, lineage).

The detailed sub-rules for steps 1-5 follow (numbered 1, 2, 2.5a, 2.5b, 3, 4 from the original pipeline):
1. **Strategy first.** Read the brand's live strategy docs and name the messaging focus this batch serves: which priority, which message, which register, which buyer awareness level. No prompts before this is written down. (RULE TWO below enforces the per-ad version.)
2. **Find reference ads LIKE THAT.** Mine references that match the chosen message, not just any winners: the search terms come from the strategy (e.g. "consolidation" strategy → mine stack/replace/simplify statics). Own-account proven ads first, then the field — and the field means the brands fighting for THIS strategy's buyer, not whatever happens to be tracked (production-verified failure: a batch anchored to tracked-set winners was rejected because those winners served other personas — a prenatal mom ad and a discount-gummy ad are not references for a premium performance buyer, however well they rank). Use `discover` mode to search beyond the tracked set, name the persona-mirror brands from the strategy docs, and subscribe the right ones so the set stays current. Every candidate reference must pass the **reference-fit scorecard** before it anchors anything — all six, not four of six:
   - Same BUYER (the ad targets this strategy's persona, not just the category)
   - Same JOB (it does the work our message needs done: consolidation ad for a consolidation message)
   - Same REGISTER (an ad that wins on hype or deep discounts is unusable for a calm premium brand even if it ranks #1)
   - Performance-PROVEN (impressions rank or sustained days running, cited)
   - Offer-MATCHABLE (we can honestly replicate its offer mechanics — a 61%-off ad is dead on arrival for a no-discount brand)
   - Compliance-SAFE to recreate (its claims can be translated into claims WE can carry)
2.5b. **The headline gate (runs after the 2.5a mining pass) — mined language is raw material; headlines are CRAFTED through the brand's headline doctrine.** (Production-verified: verbatim transplant produces weak testimonial-style copy — "Reviews are for UNDERSTANDING, not for COPYING" is the doctrine's own warning.) Before any headline ships, run it through the brand's headline docs (for this vault: `iterations.md` STATIC AD ITERATIONS section + `problem-solution-headline-writer.md` / `lifestyle-headline-generator.md`, routed via `expertise-routing.md`) and it must pass ALL of: (a) names ONE ICP and ONE emotional driver; (b) operates at emotional Level 2 (what the problem stops them doing) or Level 3 (what it makes them feel about themselves), never Level 1 surface; (c) matches customer sophistication (3+ failed solutions = acknowledge the journey, myth-bust, contrarian reframe); (d) uses a NAMED structure (rejection/contrast, parallel elements, question, metaphor, identity callout, reaction...); (e) System 1 readable in 2 seconds, under ~10 words for the headline layer; (f) uses the audience's insider words (an optimizer says "stack", a general buyer says "shelf"); (g) NO self-defeating numbers: any stat must be impressive without a calculator, and a number whose denominator undercuts the claim (117 of 2,500) is an automatic kill. Feature lines get pushed to benefit, then to emotional benefit, before they ship.

2.5a. **The language pass — the messaging must be the CUSTOMER'S, mined, not written.** Before writing a single line of copy: (a) open the brand vault's voice-of-customer bank if one exists (`personas/voice-of-customer/`: metaphors, objections, pain phrases, outcome phrases, anti-language — each entry carries source, recurrence, and confidence); (b) pull the live corpora at volume — reviews (semantic queries PLUS one SQL random-sample to enumerate the corpus's actual phrase set), ad comments, post-purchase surveys; (c) build the batch's phrase bank with epistemic labels, flagging demo-seeded or templated corpora honestly. Then every headline and subline must trace to a sourced phrase, in this priority: customer METAPHORS first (the mental model — e.g. "a shelf full of vitamins" beats any copywriter's line about consolidation), then objections the customer answers THEMSELVES (the objection-resolution arc is a complete ad), then pain and outcome phrases. Check the anti-language file last: words customers never use are words the ad can't use. A headline with no phrase lineage is a flag, not a ship.

3. **Generate with the full craft stack.** When the brand vault carries a proven template library (e.g. the 40-format `ai-static-ad-generation` doc), the template IS the prompt: fill only the bracketed variables and keep the structure verbatim — production-verified: custom-written prompts underperform the templates on copy discipline. Every selected format also gets its required "Why This Format for [brand]" justification (2-3 sentences tying it to strategy, audience, and account data) BEFORE generation. Custom reference-transfer prompts are the fallback for formats no template covers. Copy from verified customer language only + design psychology (eye path, money shot, clarity, color) + the reality pass (RULE THREE). All of it, every ad.
4. **Review and improve.** Discriminator + gates (review-gates.md), then iterate: fix what the gates flag, re-roll or post-composite surgically, and re-gate. An ad ships only after the loop, and every lesson from the loop gets written back into this skill.

**RULE ZERO — never generate a static from a blank page.** Statics generated from archetype descriptions alone come out as sparse, clean, template-y "AI posters" — instantly clockable as AI (production-verified failure: a 5-static batch built this way was rejected on sight). Every static generation MUST be anchored to a real, currently-winning reference ad:
1. Mine 2–3 top-impression statics from Parker (`search_competitor_facebook_ads` with `mediaType:'image'`, sorted by impressions rank — each ad's `imageUrl` on Parker storage is stable and importable via `media_import_url`).
2. Study the reference like a human designer would: winners are **copy-dense** (advertorial paragraphs, story, a joke) or **photo-real** (actual phone photos with UI overlays), and always slightly imperfect — blurred lines, casual punctuation, real chrome. Sparse elegant negative space is the AI tell, not the win.
3. Attach the reference as the first image with the layout-transfer instruction: *"Image 1 is a layout/typography/energy reference from a real winning ad — match its hierarchy, text density, margins, and native feel exactly; do NOT copy its brand, product, mascot, colors, or any of its words."* Attach the real packshot as the second image.
4. Compliance guard on native formats: never fabricate a consumer testimonial or endorsement text (FTC) — the text-message/DM format can dramatize the PROBLEM as a dialogue, or use verified real quotes from the vault; real winners like Ritual's use actual customer texts (note their blurred-line authenticity marks).

**RULE ONE — statics are photoreal; the animation lane never leaks in.** Statics anchor to REAL product photography (the brand's packshot import), real-environment scenes, and real-ad layout DNA. Animation-lane assets — character-face packs, stylized heroes, clay/Pixar anything — must never be used as product references for statics (production-verified failure: a video-lane character pouch leaked into a statics batch through a reused anchor id and put a face on the product in 3 of 4 renders). Keep a per-lane anchor registry in the run manifest: `packshot` (photoreal, for statics) and `character-pack` (stylized, for video) are different assets with different ids, named so a reused id can't cross lanes. Animation-STYLED statics exist only as deliberate companions to an animation video campaign, and only when the user asks for that lane.

**RULE TWO — every batch is a strategy sprint, not a grab bag.** Before writing a single prompt, read the brand's live strategy doc (roadmap/priorities) and give each static a one-line strategic purpose (which priority it serves) plus a psychology brief: the dominant mechanism (self-reference, social proof, information gap, salience, loss aversion...) and the emotional zone (default calm Zone 1 or mild Zone 3 on cold audiences). One dominant mechanism per frame; stacking fighters kills both. A technically perfect static serving a superseded strategy is a failed static.

**RULE THREE — the reality pass (photoreal statics must look shot, not designed).** Production-verified: a reference-anchored batch with perfect craft was rejected on sight for being immaculate. Native formats live or die on believable imperfection, so build it in at the prompt level: phone-grade lighting (harsh kitchen overhead, on-camera flash, mixed color temperature), casual framing (slightly crooked, off-center, edge-cropped), real clutter (counters with keys and mail, not styled props), sensor grain and mild motion blur, platform chrome where the format has it (Delivered receipts, timestamps, battery bars). Two hard sub-rules: (1) **if the headline references a visual, that visual must be IN the frame** — an ad about "the pile" shows the pile; (2) **testimonial formats require real talent assets** — never render a synthetic person under a member-quote format, pause the format until real photography exists. Polish is reserved for formats whose reference winner is polished AND whose humans are real.

**The design-psychology pass (run on every static before the reviewer sees it; full doctrine in `references/design-psychology.md`; a failure at the one-second test, the one-channel rule, or decoding-required is a REDESIGN, not a polish note):**
- **The one-channel rule** — one element carries the payoff; if the claim or joke requires reading element 3 of 5 (a table, a notification stack, distributed copy), collapse it to one channel or kill it. Production-verified: a lock-screen concept with 5 reminder cards and a handwritten T-chart were both rejected as "hard to read, takes effort" despite clean craft.
- **One-second test / eye path** — what lands first, second, third? The hook must be the unmissable first read (size, contrast, placement, isolation). If the first read isn't the hook, fix hierarchy, not copy.
- **Money shot** — can a viewer tell in half a second what the product is and how it benefits them? Type-led formats earn an exception only when the reference winner is type-led.
- **Clarity over cleverness** — if it needs interpretation, it's dead. Everyday words only.
- **Contrast strategy** — 2–3 colors total; hook = highest contrast; CTA distinctly colored; dark or vibrant fields beat white-on-white feeds.
- **Native text levers where the format calls for it** — handwritten-on-paper, Post-its, highlighter emphasis, dramatic size mixing; recognizable cultural formats (screenshots, threads, notes) buy familiarity while the content supplies novelty.

**One-shot full layout** (headline + product + visual in one generation) works on `nano_banana_pro` and `gpt_image_2` for short-copy formats (offer cards, callouts, memes, cartoon statics). Rules:
- Exact copy in quotes; name the font style per text block: `headline "YOUR BATHROOM SCALE IS LYING" in bold condensed white sans-serif at the top; subline "…" in thin grey sans-serif below`.
- Limit text blocks (1–3) — dense body copy still produces gibberish; for listicles/advertorials with paragraph text, generate the visual and set type in post (or accept platform copy fields doing the work: headline 27 chars, primary text 125 shows before truncation).
- Misspelling recovery: don't re-roll a good composition — instruct a surgical edit: "Keep everything identical; correct the headline to spell 'HYDRATE' exactly." Letter-by-letter spell odd words. OCR-proofread every output before shipping.
- Surgical-edit limit (production-verified): a "change only X" edit re-renders the WHOLE image, so an edit aimed at one tiny defect can mangle previously-perfect micro-type elsewhere. For small cleanups on flat surfaces (stray seals, ghost marks, dust), skip the model entirely: do a local clone-patch in post (PIL: copy an adjacent clean patch of the same surface, paste over the defect with a feathered mask). Deterministic, free, and it can't break the rest of the frame.
- Brand control: state hex + color name together ("#00A87E deep green"); attach the actual logo file and brand product photo as references — never generate a logo or label from memory.
- Prompt the layout's negative space deliberately: "clean negative space in the top 20% for the headline; product bottom-right at ~15% of frame."
- **Clean-plate + hooks-in-post (the cheap hook test):** for native formats whose text is an overlay (IG-story bubbles, notes lists, receipt chrome), generate the scene as a CLEAN PLATE with no text, then set the hook in post deterministically (PIL rounded-rect bubble + bold rounded sans mimics IG's text tool; SF Pro for iOS chrome). One generation yields N hook variants on an identical frame — a true hook A/B with zero extra credits — and post-set type can never garble. Production-verified companion move: composite generated photos INTO existing UI frames (text threads, notes) rather than asking the model to re-render the UI around them.
- Batch: one concept × 4 hooks × 2–3 visuals via `params.count` and prompt variants; 2–4 statics per video concept minimum.
- `ms_image` (DTC Ads model) is the alternative rig: curated ad formats + brand kits (`show_marketing_studio` type `image_style` to list styles — REQUIRED before use; type `brand_kit` and `product` for assets). Use it when the user wants Higgsfield's managed ad formats instead of hand-built prompts.

**Animation-style statics (gated — companion format ONLY):** these exist solely to accompany a running animation VIDEO campaign, at the user's explicit request — never as the default statics lane (see RULE ONE). When gated in: reuse the ad's locked style block + character bible + hero reference, compose the frame for the static's aspect ratio, and add the headline text layer in-generation (NB Pro) or in post. The villain-confession speech-bubble static ("I'm your bathroom scale. I've been lying to you for years.") is the highest-leverage companion format.

## Specs & placement (Meta, 2026)

- **Feed: 4:5 (1440×1800)** is the default winner (more screen than 1:1); keep 1:1 1080×1080 for right column/catalog; **9:16 1080×1920** for Stories/Reels.
- **Safe zones (unified March 2026):** keep critical elements out of the top 14%, bottom 20–35%, and 6% each side on 9:16; for Advantage+ auto-placements keep headline/logo/CTA inside a centered 1080×1080 square. Prompt it: "no critical elements in the top 15% or bottom 25% of the frame."
- The old 20% text-rejection rule is dead, but text-heavy images still get deprioritized delivery — keep text under ~1/3 of the image area.

## Compliance (gate every static)

- **AI disclosure is enforced (2026):** undisclosed AI creative is a top-3 Meta rejection reason. Check the AI-generated-content box in Ads Manager for photorealistic AI content; obviously-illustrated/cartoon statics get the lighter nested label — one more reason the animation-style lane is strategically safe. Don't strip C2PA metadata.
- **Supplements/health:** 18+ targeting; no weight-loss before/after side-by-sides (enforcement now catches *implied* transformation); no outcome guarantees or condition-specific claims; realistic timelines; "not intended to diagnose, treat, cure, or prevent" available in copy.
- **No personal-attribute callouts** ("Do you suffer from…?") — reframe third-person/aspirational.
- Real urgency only; competitor unnamed in us-vs-them.
- Brand claims doc from the vault wins over everything here.

## Testing cadence

Top accounts ship 12–15+ genuinely new statics weekly (spend-indexed: ~10/mo at $10k, 60–100/mo at $200k). Vary ONE variable per variant (hook, visual, or format — not all three); diversify across 5 axes over the batch: format, visual style, angle, offer, persona. Winner bar: CPA holds ±15% while scaling, survives ~14 days of saturated spend, 50+ conversions before calling it. Feed winning static hooks back into the video pipeline as the next ad's opening beat.
