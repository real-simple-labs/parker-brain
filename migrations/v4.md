# v4 — the fast update: script-driven re-sync of the copied layer

**Applies to:** brains pinned at `v3` or earlier (after prior migrations). No-op on `v4+`.

**What changes:** the manual per-file re-sync of the copied executable layer is replaced by a deterministic script, `parker-system/scripts/sync-executable-layer.py`. It applies the three piles mechanically — untouched copies refresh silently, team-edited files stay theirs and are listed, nothing is deleted — so a routine update becomes one question and one short summary.

## Steps

1. **Run the sync script** from the brand repo root, with the pin the brain moved *from*:

   ```
   python3 parker-system/scripts/sync-executable-layer.py --from <old pin>
   ```

   This refreshes every unmodified file the build copied out of the mount — the craft skills, routine skills (including `/update-brain` itself, which is how this new behavior arrives), review-gate agents, hooks, checker scripts, and schedule recipes. Do not hand-compare any of those files; the script's report is the outcome. Skip this step only if the report shows it already ran for this bump (re-running is a safe no-op).

2. **Add the "where brand adaptations live" note to the brand's `CLAUDE.md`** — this edits brand-authored content, so show it to the team before applying, per the offer rules. If `CLAUDE.md` already says adaptations belong in brand docs first, skip. Otherwise, in the section that tells the agent to refine or adapt the skills to the brand, append:

   > When a skill needs adapting to this brand, put the adaptation in the brand's own docs first — `brand-lens.md`, the brand rules, the script-voice profile, running notes — because the skills read those at runtime and factory updates never touch them. Edit a copied skill or schedule file directly only when the adaptation truly can't live in a brand doc; a file you edit is yours from then on, and updates will leave it frozen at your version (you'll see it listed as "left alone" in each update summary).

3. **Record it:** note migrations applied through `v4` in `running-notes/standard-sync.md` and set `parker_brain_version` to `v4` in `parker_config.json`.

## Verify

- `python3 parker-system/scripts/sync-executable-layer.py --from v4 --dry-run` runs clean and reports nothing to refresh.
- `.claude/skills/update-brain/SKILL.md` contains the "Keep it quick" section.
