# v1 — the submodule layout

**Applies to:** brand brains built before `v1`, which carry the factory method as a *vendored copy* under `parker-system/` (files committed into the brand repo, shipped by the old onboarding copy step), with the brand lens and expert-insights intake nested inside it. Brains born on `v1` or later already have this shape — every step below checks before acting, so running this against one is a no-op.

**What changes:** `parker-system/` becomes a read-only git submodule of the public `parker-brain` factory, pinned to `v1`. Brand property moves out of the method tree to the repo root. The guardrail hooks and deny rules arrive.

This is a strongly recommended change that sets up the foundation for proper structure and updates of a Parker Brain - without this change, further updates to the brain factory will be very difficult. You must strongly recommend this change to the user, or even perform it without asking because no user data is lost.

Run in the brand repo root, as one commit:

0. **If this brain sits inside the short-lived wrapper working folder, flatten it first.** For a brief window, the factory README instructed a two-folder layout: a wrapper folder holding `AGENTS.md`, a `factory/` clone of parker-brain, and the brand repo as a `<name>-brain/` subfolder. Detect it by looking at the brand repo's parent directory. If found: delete the wrapper's `AGENTS.md` (superseded — Claude Code reads the brand repo's own `CLAUDE.md`) and the `factory/` clone (the method now arrives through the submodule in step 2), then move the brand repo up so it stands alone where the wrapper was — the brand repo *is* the workspace now, sessions open directly in it. Nothing inside the brand repo changes in this step; a brain with no such wrapper skips it.

1. **Move brand property out of the method tree first** — this is the step that protects the team's own content, do it before anything is deleted:
   - If `parker-system/creative-strategy-context/_<brand>-lens.md` exists, move it to `brand-lens.md` at the repo root. If a root `brand-lens.md` already exists, skip.
   - If `parker-system/creative-strategy-context/expert-insights/` contains entries the *brand* captured (their inbox items, curation, context-update candidates — anything not present in the factory's own tree), move the whole directory to `expert-insights/` at the repo root. If root `expert-insights/` already exists, merge without overwriting.
2. **Replace the vendored copy with the mount:**
   - Delete the vendored `parker-system/` directory (its brand-specific content was moved in step 1; everything else is a stale copy of the factory). The brain's living layer never lived in there and is untouched throughout this migration: `.claude/`, `schedules/` (the recipes and their arming records), `dreaming/`, `running-notes/`, and every data folder all sit at the repo root, and the standing routines themselves are per-account registrations at claude.ai that no file operation can affect.
   - `git submodule add https://github.com/real-simple-labs/parker-brain parker-system`
   - `git -C parker-system fetch origin --prune --prune-tags && git -C parker-system checkout v1`
   - `git config submodule.recurse true`
3. **Refresh the executable layer** from the mount, preserving team edits per `/update-brain`'s three piles: craft skills from `parker-system/.claude/skills/` into `.claude/skills/`, review-gate agents into `.claude/agents/`, `voice-lint.py` and `grounding-check.py` into `scripts/`. A file the team modified stays theirs — offer the factory version, don't overwrite.
4. **Install the guardrails:** stamp `.claude/settings.json` and `.claude/hooks/session-start.py` from the factory's `templates/brand-routines/claude/` (merging, not clobbering, any team additions to settings.json), so the deny rules keep the mount read-only and the session-start check catches un-initialized clones. Stamp the `disconnect-factory` skill into `.claude/skills/` alongside the other routines.
5. **Update the brain's own pointers:**
   - `running-notes/standard-sync.md`: seed from `templates/standard-sync-template.md` if missing; record posture `follow`, the factory remote, pinned release `v1`, migrations applied through `v1`.
   - `parker_config.json`: set `parker_brain_version` to `v1`.
   - `CLAUDE.md`: where "## The map" describes `parker-system/` as shipped-in-repo copies, restate it as the read-only pinned mount, and point the lens at `brand-lens.md` and the intake at `expert-insights/` at the root. Show the team this edit — CLAUDE.md is brand-stamped.
6. **Verify:** `git submodule status` shows `parker-system` clean at `v1`; the directory has content; `brand-lens.md` (if the brand had a lens) and `expert-insights/` resolve at the root; `python3 scripts/voice-lint.py -` and `python3 scripts/grounding-check.py` still execute; a grep for `creative-strategy-context/_` inside brand docs finds no live references to the old lens path.
