# v6 — simpler git flow, pull-first and push-always enforcement

**Applies to:** brains pinned at `v5` or earlier (after prior migrations). No-op on `v6+`.

**What changes:** the git procedure loses its token-in/token-out ceremony — credentials now just live in the remote, and an auth error means a two-line refresh (`setup_parker_brain` → `git remote set-url origin <fresh url>`) — the session-start hook now runs the start-of-session pull itself when the tree is clean, and the commit-and-push-immediately discipline (with the explicit-"don't commit" user override) is stated as an absolute in `save-brain` and the brand `CLAUDE.md`. The updated skill and hook live in the copied bundle, so the pin bump's re-sync delivers them.

## Steps

1. **Refresh the "How this brain saves itself" section in the brand's `CLAUDE.md`** — brand-authored content, so show it to the team before applying. Replace the section's body with the section of the same name in `parker-system/templates/brand-brain-CLAUDE-template.md` (it now teaches the simpler credentials-in-the-remote flow and the two absolutes: pull first, commit-and-push immediately without asking). If the section is missing entirely, add it.

2. **Record it:** note migrations applied through `v6` in `running-notes/standard-sync.md` and set `parker_brain_version` to `v6` in `parker_config.json`.

## Verify

- `.claude/skills/save-brain/SKILL.md` says credentials live in the remote (no `set-url` bracket around every push).
- `.claude/hooks/session-start.py` runs the pull itself (contains `attempt_pull`).
- The brand `CLAUDE.md` section carries the two absolutes (pull first; commit and push immediately, never asking — with the explicit user "don't commit" as the only override).
