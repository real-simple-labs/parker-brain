# v5 — the git procedure: save-brain, git-guard, and credential rules

**Applies to:** brains pinned at `v4` or earlier (after prior migrations). No-op on `v5+`.

**What changes:** the brain gains a canonical way to talk to GitHub — the `/save-brain` skill (Parker's short-lived credentials via `setup_parker_brain`, explicit `git push origin main`, token never left in the saved remote, submodules always, conflicts best-effort, no `gh`, no force-push, with a self-hosted exception detected from the origin URL) — and a `git-guard` PreToolUse hook that blocks the wrong git moves on managed repos and teaches the right ones. The skill and hook travel in the copied bundle, so the pin bump's re-sync delivers them; this migration handles the two files the sync can't rewrite.

## Steps

1. **Add the "How this brain saves itself" section to the brand's `CLAUDE.md`** — brand-authored content, so show it to the team before applying. Skip if a section by that name already exists. Copy it verbatim from the factory template's section of the same name in `parker-system/templates/brand-brain-CLAUDE-template.md`.

2. **Record it:** note migrations applied through `v5` in `running-notes/standard-sync.md` and set `parker_brain_version` to `v5` in `parker_config.json`.

## After the re-sync (verify and patch, not a migration step)

The post-migration re-sync delivers `.claude/hooks/git-guard.py`, `.claude/skills/save-brain/`, and — when the team never customized it — the updated `.claude/settings.json` that registers the guard. After the sync report, check `.claude/settings.json`: if it does not register `git-guard.py` under a `PreToolUse` hook with matcher `Bash` (the sync left the file alone because the team changed it), merge this entry into its `hooks` block, showing the team first:

```json
"PreToolUse": [
  {
    "matcher": "Bash",
    "hooks": [
      { "type": "command", "command": "python3 .claude/hooks/git-guard.py" }
    ]
  }
]
```

Keep the team's existing entries untouched, and do **not** add the `2>/dev/null || true` wrapper the other hooks use — the guard's exit code and error text are how it works, and it fails open on its own.

## Verify

- `.claude/hooks/git-guard.py` and `.claude/skills/save-brain/SKILL.md` exist (the re-sync brought them).
- `.claude/settings.json` registers the guard under `PreToolUse`.
- In a managed repo, `echo '{"tool_name":"Bash","tool_input":{"command":"gh pr list"}}' | python3 .claude/hooks/git-guard.py` prints the teaching message and exits 2.
- The brand `CLAUDE.md` contains "How this brain saves itself".
