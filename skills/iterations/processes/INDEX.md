# Iterations — Process Index

This index lists every iteration process Parker can run. Parker reads this before strategy.md picks which processes to load. Only the chosen process files are pulled into context — the rest stay out.

Each entry: process name → what it does → when to pick it. Best-for and worst-for live in `strategy.md` to keep this index light.

## High-confidence processes

- **frankenstein-stitch** — Lift a proven hook from one winning ad onto the body of another. Pick when multiple winners exist and a clean hook-to-body pairing is identifiable.
- **creator-reshoot-same-script** — Send the proven script to a new creator. Pick when script is converting and the creator is fatiguing or a new demographic needs testing.
- **format-swap** — Rebuild the winning message in a different format. Pick when the message converts and the concept has not been tested across formats yet.
- **human-desire-swap** — Swap the human desire the ad activates while keeping product, format, and structure intact. Pick when the current desire is identifiable and the brand has not tested other emotional engines.
- **mashup** — Combine best moments from multiple winners into a compilation. Pick when multiple winners exist and the moments flow together.
- **hook-iteration** — Modify the first three seconds while preserving the converting body. Pick when CTR or hook rate is the bottleneck and the body works.

## Process menu

- **messaging-angle-iteration** — Rewrite around a different angle, awareness level, or framing. Pick when visuals work but the message is not converting.
- **audience-iteration** — Demographic, authority, voiceover, or persona-variant swaps beyond same-script reshoots. Pick when expanding into a new audience segment.
- **static-headline-iteration** — Test a new headline structure on a static ad. Pick when the headline is the bottleneck on a static.
- **edit-pacing-iteration** — Low-risk edit refreshes — pacing, b-roll, captions, music, headline overlays. Pick as a quick-win complement to a bigger swing.

---

Strategy may select a single process or two-to-four processes for a multi-iteration recommendation set. Each picked process file owns its own execution playbook.
